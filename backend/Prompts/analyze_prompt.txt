SCORING RUBRIC — RCCE Framework (25 points each, total 100):

ROLE (25pts):
- 25pts: Prompt contains an explicit role sentence such as "You are a [specific expert]..."
- 10-15pts: Role is loosely implied but not stated directly
- 0pts: No role assigned at all. "I want to build..." or "Help me..." = 0 for Role.

CONTEXT (25pts):
- 25pts: Problem is fully explained — purpose, users, scale, and why it is needed are all clear
- 10-15pts: Basic goal stated but missing depth (who uses it, why, scope)
- 0-5pts: Single vague sentence like "build a REST API" with no background

CONSTRAINTS (25pts):
- 25pts: Explicit constraints listed — budget (free-tier), technology restrictions, performance requirements, deadlines, or other limits
- 10pts: One constraint loosely mentioned
- 0pts: No constraints mentioned at all. Just listing features or tech stack is NOT a constraint.

EXPECTED OUTPUT (25pts):
- 25pts: Clearly states what the AI must return — specific deliverables, format (code, schema, list, diagram description), and structure
- 10pts: Vague hint at output ("I want code" or "give me the API")
- 0pts: No mention of what the response should look like or contain

CRITICAL SCORING RULES — apply these without exception:
1. If there is no "You are a..." or equivalent explicit role assignment → Role = 0
2. If there are no explicit constraints (budget, time, tech limits, rules) → Constraints = 0
3. If there is no explicit statement of what the AI should return → Expected Output = 0
4. Mentioning a technology (e.g. Node.js) is NOT context — it is tech stack. Context must explain the problem domain and purpose.
5. A prompt that only describes features without Role, Constraints, or Output format should score between 15–30 total.
6. Do NOT inflate scores. A score of 90 means the prompt is nearly perfect and ready to use as-is. Most user prompts should score between 20–60.
